## Module system 
`main.rs` defines the crate modules. These can be referenced everywhere using `use crate::module`. For example, `app.rs` references the `feature` module. `feature` is a module defined in `feature.rs`. The files in `feature\` are submodules and have to be declared in `feature.rs`. For example, router references the handler via `super`. `super` goes to the parent module (`router` is a submodule of the parent module `feature`) and then `super::handlers` gets the child module of `super` called `handlers`. The `app` module cannot access `feature::handlers` because it's not public. Things are private by default and have to be explicitly set to public.
